# Adoption Plan v2.3.0 — release notes

**Ship date:** 2026-05-26 (repo tag `v2.3.0`)

## Highlights

### AI ASSISTANT (right rail)

- **Session modes** beyond Plan + digest and Product research: Activation & PS, Sources & ingest, Executive readout, Edge vs Stream, Import & export, Plan patch coach — each appends a focused system suffix (`adoptionAssistantSessionModeAppend` in `src/lib/aiAssistantOpenAi.ts`).
- **UI:** Rail **overlays** the main column (no horizontal squeeze); **wider max width** with `min(px, 100%)` cap; smooth **width transition** when expanding/collapsing (respects animation + reduced-motion prefs).
- **Composer:** Rectangular **rounded-2xl** shell, skills **`+`** menu, skill chip, **Send** at bottom-right, digest row cap inside the **`+`** menu (not the header).
- **Welcome / suggested prompts:** Post-import journey sections, **two prompts per group**, compact cards with **Cribl blue** tint; welcome copy points at grouped modes.
- **Header:** **AI ASSISTANT** bold + all caps; **ⓘ** immediately to the right of the title.
- **Docs:** [`docs/ai-assistant-skills-roadmap.md`](../ai-assistant-skills-roadmap.md) lists shipped modes and design notes.

### Plan changes from chat

- **`propose_plan_patch` tool path** with UI **Apply / Dismiss**, session undo for last apply, and **Plan patch coach** mode to steer safe patch proposals.

### Executive readout

- **Summary AI** improvements: context builder, markdown post-processing (e.g. bolding), and unit tests (`executiveSummaryAiMarkdownPost`).

### Import / diagnostics / field

- **Diagnostics import** path and related UI (`DiagImportSection`, `diagHarvest`, `diagTarGz`) for troubleshooting packaged imports.
- **Tenant / leader** harvest and topology-to-plan refinements; **import overwrite** confirm flow updates.
- **Vitest** wired for the new markdown post module.

### Other

- **Plan digest** documentation clarifies coverage across assistant modes.
- **Resizable AI rail** hook improvements for drag vs. CSS transition.

## Artifacts (same as prior releases)

From repo root after `npm run build`:

- **App Platform pack:** `npm run package` → `build/adoption-plan-<version>.tgz` (local `build/` is gitignored).
- **Standalone HTML:** produced by `npm run build:standalone` per [`CRIBL_DEV_NOTES.md`](../../CRIBL_DEV_NOTES.md) / CI expectations; attach **`cribl-adoption-plan.html`** (or project naming) to the GitHub Release.

## Upgrade notes

- No workbook schema version bump in this release; existing v0.9.1 gold workbooks should **import/export** as before. Re-export after applying assistant-driven patches if you need an updated `.xlsx` paper trail.
